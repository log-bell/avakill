Getting Started

Introduction

AvaKill is the open-source safety firewall for AI agents. It intercepts tool calls in real-time, enforces YAML policies, and blocks dangerous operations before they execute.

Whether you are:

  • a developer looking to add guardrails to your AI agent
  • a team lead looking to enforce safety policies across agents
  • a security engineer looking to audit and monitor agent behavior

AvaKill has you covered. One YAML policy protects every agent with sub-millisecond overhead.

Key features

AvaKill provides a comprehensive toolkit for controlling AI agent behavior:

  • Tool-call interception — evaluates every tool call against your policy before execution
  • Native agent hooks — drop-in support for Claude Code, Gemini CLI, Windsurf, and OpenAI Codex
  • YAML policies — human-readable rules with glob patterns, arg matching, and rate limits
  • Complete audit trail — every decision logged to SQLite with full arguments and latency
  • Framework wrappers — GuardedOpenAIClient, GuardedAnthropicClient, LangChain callback handler
  • Sub-millisecond latency — pure rule evaluation, no ML models in the critical path

What would you like to do?

Choose your path based on how you want to integrate AvaKill:

⚡
Quickest start: Run pipx install avakill && avakill guide to launch the interactive setup wizard. It handles policy creation, daemon startup, and hook installation in one flow.

Protect AI coding agents (Claude Code, Gemini CLI, Windsurf, Codex) — start with installing hooks. No code changes to your agent required.

Add guardrails to your Python code — use the @protect decorator or call Guard.evaluate() directly.

Enforce policies across a team — read the Policy Cascade docs for multi-level policy discovery and merge.

Installation

Install AvaKill globally with pipx (recommended) so it's available across all your projects:

shell 
pipx install avakill

Or install into a virtualenv with framework extras:

shell 
pip install "avakill[openai,anthropic,langchain]"
⚠️
macOS users: Use pipx or a virtualenv. macOS restricts pip install to the system Python.

Set up your policy

Run the interactive setup wizard to create your first avakill.yaml policy:

shell 
# Interactive 5-step setup: choose template, configure rules, write YAML
avakill setup

# Or non-interactive with a template
avakill setup --template strict --non-interactive

Templates

AvaKill ships with built-in templates: strict (deny-by-default), standard (balanced), and permissive (allow-by-default with blocklist). Use --template to start from one.

Validation

Validate your policy file at any time:

shell 
avakill validate              # validates ./avakill.yaml
avakill validate -f custom.yaml

Add AvaKill to your code

The simplest integration is the @protect decorator. Wrap any function that makes tool calls:

python 
from avakill import protect

@protect("file_write")
def write_file(path: str, content: str):
    Path(path).write_text(content)

For more control, use Guard.evaluate() directly:

python 
from avakill import Guard

guard = Guard()
decision = guard.evaluate("shell_execute", {"command": "rm -rf /"})

if decision.action == "deny":
    print(f"Blocked: {decision.reason}")

Protect AI coding agents

AvaKill integrates with AI coding agents via native hooks. No code changes required — just install and go.

Start the daemon

The daemon provides a long-running evaluation service over a Unix domain socket:

shell 
avakill daemon start

Install hooks

Hooks register AvaKill into your agent's configuration. One command per agent:

shell 
avakill hook install claude-code
avakill hook install gemini-cli
avakill hook install windsurf
avakill hook install openai-codex

Standalone mode

If you don't want to run a daemon, hooks can evaluate policies directly in standalone mode. Slightly higher latency per call, but no background process required.

Monitor & debug

Every tool call is logged to a local SQLite database with the decision, matched policy, latency, and full arguments.

shell 
# View recent audit events
avakill logs --last 20

# Follow logs in real-time
avakill logs tail

# Launch the terminal dashboard
avakill dashboard

# Test a tool call from the CLI
avakill evaluate shell_execute --args '{"command": "ls -la"}'

SDKs & framework wrappers

AvaKill provides drop-in wrappers for popular AI frameworks:

  • OpenAI — GuardedOpenAIClient wraps the OpenAI client
  • Anthropic — GuardedAnthropicClient wraps the Anthropic client
  • LangChain — AvaKillCallbackHandler integrates via LangChain callbacks
  • MCP — MCPProxyServer provides transparent tool-call interception
